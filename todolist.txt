3. Windows 환경에서는 종종 문제 발생

Windows에서는 multiprocessing이 spawn 방식이기 때문에,

num_workers > 0일 경우, worker 프로세스가 새로운 Python 프로세스를 실행해야 하며

코드가 if __name__ == '__main__': 안에 없으면 worker가 무한 재귀적으로 실행되면서 오류가 납니다.

추론 코드에서 num_workers=0으로 설정하면 이런 문제를 방지할 수 있습니다.
